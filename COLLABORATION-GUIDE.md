# Group Collaboration Guide

> 群协作记忆系统使用指南

## 📖 概述

Harness 提供群协作记忆能力，支持在群聊场景下：
- 追踪话题讨论
- 识别群成员
- 跟进任务分配
- 定期汇报总结

---

## 🎯 适用场景

| 场景 | 描述 |
|------|------|
| **项目协作群** | 多人讨论项目，需要记录决策和待办 |
| **话题讨论** | 使用 #话题 标签组织讨论 |
| **任务分配** | @成员 分配任务，自动跟踪进展 |
| **定期汇报** | 按天/周总结讨论和工作进展 |

---

## 📁 Wiki 结构

```
memory/wiki/
├── groups/
│   └── <群名>.md        # 群组信息和成员列表
├── topics/
│   └── <话题名>.md      # 话题讨论记录
└── people/
    └── <人名>.md        # 成员信息（可选）
```

---

## 🏷️ 话题格式

### 使用 #话题 标签

```
#DobiAi 品牌讨论
#HarnessDev 开发进展
#ProjectX 项目 X
```

### 话题页面模板

```markdown
# #话题名

**状态:** 🟢 进行中 / 🟡 等待 / ✅ 完成
**创建:** YYYY-MM-DD
**负责人:** @某人
**参与者:** @成员 1, @成员 2, @成员 3

## 背景
话题的初始背景和目的

## 讨论记录
- YYYY-MM-DD @成员 1：建议...
- YYYY-MM-DD @成员 2：同意，但是...
- YYYY-MM-DD @成员 3：多比建议...

## 决策
- [x] 已决策 1
- [ ] 待决策 2

## 待办
- [ ] 任务 1 (@成员 1)
- [ ] 任务 2 (@成员 2)
- [ ] 任务 3 (@多比)

## 相关
- [[相关 wiki 页面]]
```

## 📊 汇报规则

### 任务汇报

| 情况 | 汇报方式 |
|------|---------|
| **@分配任务** | 及时汇报进展和结果，@安排人 |
| **话题讨论** | 按天/周汇报（根据活跃度） |
| **汇报内容** | 自己的工作 + 群成员讨论总结 |

### 汇报模板

```markdown
## 📋 工作汇报（YYYY-MM-DD）

### 多比的工作
1. **任务 1** - ✅ 完成
   - 结果：...
   
2. **任务 2** - 🟡 进行中
   - 进展：...
   - 下一步：...

### 群讨论总结
1. **#话题 1**
   - 讨论：...
   - 决策：...
   
2. **#话题 2**
   - 待办：...

### 待跟进
- @成员 1 的任务
- @成员 2 的任务
```

## 🛠️ 工具使用

### Wiki Ingest（摄入讨论记录）

```bash
# 摄入会议记录
npm run wiki:ingest meeting-notes.md

# 摄入聊天记录
npm run wiki:ingest chat-log.md
```

### Wiki Query（查询话题）

```bash
# 查询话题
npm run wiki:query "#DobiAi 的讨论"

# 查询成员相关
npm run wiki:query "@皮爷 分配的任务"
```

### Wiki Lint（检查健康状态）

```bash
# 检查话题状态
npm run wiki:lint

# 检查过时话题
npm run wiki:lint --stale
```

## 📋 最佳实践

### 1. 话题管理

- ✅ 使用 #话题 标签开始讨论
- ✅ 重要决策记录到话题页
- ✅ 定期关闭已完成话题
- ❌ 避免话题过多（定期清理）

### 2. 任务跟进

- ✅ @成员 明确分配任务
- ✅ 记录到话题页的待办列表
- ✅ 定期更新进展
- ❌ 避免任务石沉大海

### 3. 汇报频率

| 群活跃度 | 汇报频率 |
|---------|---------|
| 高（每天讨论） | 每天汇报 |
| 中（每周讨论） | 每周汇报 |
| 低（偶尔讨论） | 按需汇报 |
